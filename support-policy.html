Release Channels, Support & Early Access Policy

CoreToolsX is free to download and use.

Public stable releases are published on Modrinth using the 2026.1.x version format. These releases are the recommended versions for production servers and include stable, tested, public updates.

Paid memberships do not sell access to the stable plugin itself.

Paid memberships provide optional support services, such as priority support, setup assistance, configuration review, compatibility troubleshooting, direct debugging help, and access to experimental preview builds.

Supporter Preview Builds

Supporters may receive temporary access to preview builds, release candidates, staging builds, and non-critical hotfix candidates before they are included in a public stable release.

These builds use the 2026.1.x.y version format.

Example:

  • 2026.1.4 - public stable release on Modrinth
  • 2026.1.4.1 - supporter preview / validation build
  • 2026.1.4.2 - supporter preview / release candidate
  • 2026.1.5 - next public stable release on Modrinth

Preview builds are intended for early testing and validation. They may contain bugs, incomplete changes, experimental fixes, or behavior that can still change before the next stable release.

They are not considered supporter-exclusive stable releases.

Critical Fix Policy

Critical fixes are not locked behind paid membership.

Security fixes, exploit fixes, data-loss fixes, major crash fixes, and other critical stability fixes are released publicly as soon as they are confirmed and ready.

For serious issues, a 2026.1.x.y preview build may be shared with supporters or testers only for short validation. Once the fix is confirmed, it is released publicly as the next 2026.1.x stable version without intentional delay.

In short:

  • Stable releases are public.
  • Critical fixes are public.
  • Security fixes are public.
  • Exploit fixes are public.
  • Data-loss fixes are public.
  • Major crash fixes are public.
  • Supporter preview builds are used for testing, validation, and early feedback.

What Paid Memberships Include

Paid memberships may include:

  • Priority support
  • Faster response priority
  • Configuration review
  • Setup assistance
  • Compatibility troubleshooting
  • Debug/log analysis
  • Access to supporter Discord channels
  • Access to experimental preview builds
  • Access to release candidates
  • Access to non-critical hotfix candidates
  • Early development updates
  • Roadmap previews
  • Feedback and testing participation

Paid memberships do not provide exclusive access to the public stable plugin.

Discord Authentication

The Discord authentication bot is used only to verify active Patreon membership and assign Discord roles.

These roles may grant access to supporter-only Discord channels, priority support areas, and preview-build testing channels.

The plugin itself does not require Patreon authentication to run.

Patreon membership does not unlock in-game features, commands, modules, gameplay advantages, or server-side functionality inside the plugin.

If a membership ends, access to supporter Discord benefits may be removed, but the public stable plugin remains usable.
